Google Cloud 서비스 계정

① Google Sheets API 활성화

브라우저에서 console.cloud.google.com → 프로젝트 선택
APIs & Services → Enabled APIs → Google Sheets API 활성화 확인

② 서비스 계정 JSON 키 다운로드

IAM & Admin → Service Accounts → 기존 계정 선택 (또는 신규 생성)
Keys 탭 → ADD KEY → Create new key → JSON
다운로드된 .json 파일을 .secrets\ 폴더에 저장 (다음 슬라이드)
⚠️

API 키만으로는 Google Sheet에 쓸 수 없습니다.
서비스 계정 JSON이 반드시 필요하며, 시트에 편집자로 공유되어야 합니다.

💡 알아두기

서비스 계정 = 사람 대신 API를 호출하는 봇 계정
JSON 파일 안에 private key가 포함됨 → 절대 커밋/공유 금지
GOOGLE_APPLICATION_CREDENTIALS에 JSON 파일 경로를 지정해야 인증됨

각 탭이 저장하는 것

meta_ad_counts

날짜별/경쟁사별 Meta 광고 총개수 저장
Google Chat의 [Meta 광고] 숫자 기준 탭
displayed_meta_count 기준 — 상세 row 개수와 다를 수 있음

meta_ad_start_dates

Meta 광고별 library_id, 광고 URL, 게재 시작일 저장
30일 초과 광고 판단 기준 탭
노란색 표시 = 30일 초과 장기 집행 광고 (수동 검토 대상)

raw_supply_snapshots

경쟁사별 공개방 수 수집 결과 저장
Airbnb, LiveAnywhere, Encostay, 삼삼엠투 (건물유형별 합산)
실패 시 status / error_message 컬럼 확인

policy_updates

정책/공지 변경 감지 결과 저장
새 공지 또는 내용 변경 감지 시 row 추가
Google Chat에는 변경이 있을 때만 표시

app_versions

앱스토어/구글플레이 버전, 업데이트 날짜, 릴리즈노트
앱 설명 원문, hash, 변경 여부도 저장
is_changed = 앱 버전/업데이트 변경
is_description_changed = 앱 설명 변경
Google Chat에는 변경이 있을 때만 표시

run_log

전체 실행 결과 로그 저장
실행 성공/실패, 부분 실패, 에러 메시지 확인용
문제가 생기면 가장 먼저 확인하는 탭

GOOGLE_APPLICATION_CREDENTIALS 경로가 올바른지 확인
Google Cloud에서 Google Sheets API 활성화 여부 확인

🟡 Sheets 429 Quota 오류

API 쓰기 요청이 60회/분을 초과할 때 발생
자동 retry 로직(최대 3회, 20/40/80초 대기)이 내장됨
계속 발생하면 20~30분 후 재실행

python scripts/setup_airdna_session.py 재실행

🔴 Playwright 브라우저 없음

Executable doesn't exist 에러

✅ 핵심 명령 7단계 요약 (Windows)

# 1. 실행 정책 (최초 1회)
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
# 2. 가상환경
python -m venv .venv
.\.venv\Scripts\Activate.ps1
# 3. 의존성 & Playwright
pip install -r requirements.txt
python -m playwright install chromium
# 4. JSON → .secrets\ 5. .env 작성 6. AirDNA
python scripts/setup_airdna_session.py
# 7. 첫 실행
python -m src.jobs.run_daily
